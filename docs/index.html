HN Reader

Privacy Policy

Last updated: June 8, 2026

HN Reader is a client for reading Hacker News. This policy explains what data the app handles.

Data Collection

HN Reader does not collect, store, or transmit any personal information. There are:

  • No user accounts or sign-in
  • No analytics or crash reporting SDKs
  • No advertising or tracking
  • No data sent to the developer's servers

Local Storage

The app stores your preferences (theme, font size, font family, accent color) and visited article history on your device only using the browser's localStorage or the platform's native storage. This data never leaves your device. You can clear it at any time from the Settings page.

Network Requests

To display content, the app makes requests to these third-party services:

  • news.ycombinator.com — Hacker News website, used to determine comment ordering and display status
  • hacker-news.firebaseio.com — Official HN Firebase API for story lists and item data
  • hn.algolia.com — HN Search API for comment content

These requests are necessary for the app to function. The services may log standard HTTP metadata (IP address, user agent) per their own privacy policies. No personal information is sent alongside these requests.

CORS Proxy

On the web version, the app may use a CORS proxy to fetch article content. By default this uses api.allorigins.win. You can configure a custom proxy in Settings.

Changes

If this policy changes, updates will be posted here. Continued use of the app constitutes acceptance of the current policy.
